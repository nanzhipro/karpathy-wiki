# Project Snowballs

Karpathy's advice for early-career researchers and engineers: **pick one small, honest thing and let it compound.** Each "snowball" project is a small artifact (a script, a repo, a blog post) that, when rolled down the hill of public attention and follow-up work, accretes more mass than the original effort justified.

## Source

[Karpathy, Berkeley SkyDeck Keynote](../sources/berkeley-ai-hackathon-2024-keynote.md) — the advice segment of the pitch-night keynote.

## The pattern

1. Build a thing small enough to finish in a weekend.
2. Make it honest: the code runs, the post explains, nothing is hand-waved.
3. Ship it publicly — repo, blog, talk.
4. Iterate on it as you learn more; don't start fresh every time.
5. The compounding comes from returning to the same object with a better version of yourself.

## Karpathy's own snowballs (implied from his body of work)

- [[micrograd]] → teaching autograd → [[zero-to-hero]] series
- [[nanogpt]] → transformer pedagogy → [[llm-c]] → [[nanochat]]
- X posts → viral phrases ([[vibe-coding]], "99 percent of attention is compute bound") → research vocabulary

Each of these started tiny and accumulated reach because Karpathy kept pushing on the *same* artifact rather than starting a new one.

## Why this works

- Returns are non-linear in sustained attention on one artifact.
- Reputation accretes to identifiable objects, not to diffuse output.
- The first 80% of understanding comes from building it; the next 20% comes from explaining it; the *cumulative effect* comes from returning to it with new eyes.

## Contrast

- **Not:** "work on one big thing for ten years."
- **Is:** "finish small things, revisit them, let them grow."
