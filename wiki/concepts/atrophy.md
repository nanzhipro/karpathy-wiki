# Atrophy

Karpathy's observation (Jan 27 2026) that sustained [[agentic-engineering|agentic coding]] gradually erodes the ability to write code manually. Not a complaint — a neutral, lived-experience report.

## Source quote

> "I've already noticed that I am slowly starting to atrophy my ability to write code manually. **Generation (writing code) and discrimination (reading code) are different capabilities in the brain.** Largely due to all the little mostly syntactic details involved in programming, you can review code just fine even if you struggle to write it."

— Karpathy, Jan 27 2026

## The core distinction

| Skill | What it is | Fate under agent-driven coding |
|-------|-----------|--------------------------------|
| **Generation** | Writing code from scratch | Atrophies |
| **Discrimination** | Reading, reviewing, judging code | Preserved / sharpened |

This is cognitively non-trivial: the two skills are dissociable. You can become a sharper reviewer while becoming a weaker writer.

## Why this matters

- **Hiring / evaluation:** whiteboard-style code-generation interviews test a skill that matters less.
- **Pedagogy:** [[ramps-to-knowledge|ramps]] must front-load generation even more deliberately, because the default practice loop no longer builds it.
- **Oversight quality** depends on **discrimination**, which is what [[agentic-engineering]] actually trains.

## Open questions

- Is atrophy recoverable, and on what timescale?
- Does deep technical expertise (mentioned in Feb 25 as an "even bigger multiplier") protect against atrophy, or is it orthogonal?
- When interviewing / teaching a generation that **never** built the generation skill, what proxy works?
